## 1. 产品概述

### 1.1 一句话定位

**Lull 是一台「声音电台」：拧动复古调频拨盘在夜间频道间穿梭，每个频道是一套助眠/专注混音，整屏画布随之实时生长出对应的气象。**

> 交互隐喻是电台（调频拨盘 + 频道），视觉内核仍是生成式气象场景——电台化只改变「怎么调音」，不改变「屏幕上长出什么」（画面仍是混音的镜像，见 1.2）。

### 1.2 背景

白噪音/环境音应用是成熟品类（Noisli、A Soft Murmur、moodist、潮汐等），但普遍是「一排开关 + 一张静态背景图」。Lull 的差异化是**混音驱动的生成式场景**：雨声带来雨丝、海浪带来潮汐波带、鸟鸣迸出金色火花、雷雨触发闪电——画面是混音的镜像，不是装饰。

同时坚持**本地优先**的工程哲学：纯前端、零运行时依赖、单文件可双击运行、无账号、无埋点、数据只存 localStorage。

### 1.3 设计原则（约束所有需求）

1. **睡眠优先**：任何功能不得引入影响入睡的元素（突兀音量跳变、亮屏干扰、循环接缝）。
2. **本地优先**：无后端、无账号、无第三方请求；离线可用是目标而非选配。
3. **单文件克制**：运行时零依赖；构建工具只服务开发体验，产物仍是可双击打开的静态文件。
4. **版权洁癖**：所有音频素材必须 CC0/可商用，逐文件登记于 `AUDIO_CREDITS.md`。

## 2. 目标用户与场景

### 2.1 用户画像

| 画像 | 描述 | 核心诉求 |
|---|---|---|
| 睡眠困难者（主） | 入睡慢、易被环境音吵醒，睡前手机放床头 | 稳定不间断的声音、定时关闭、锁屏后继续播放、暗色不刺眼 |
| 专注工作者（次） | 开放办公室/在家办公，用环境音屏蔽干扰 | 快速调出熟悉的混音、长时间播放不疲劳、不占心智 |
| 冥想/放松用户（次） | 短时段放松、呼吸练习 | 颂钵等冥想音色、画面氛围感 |

### 2.2 核心场景

- **睡前**（主场景，移动端为主）：躺床上打开 → 调出「雨夜」混音 → 设 30 分钟定时 → 锁屏 → 声音持续、到点渐隐停止。
- **专注**（桌面为主）：开工时一键恢复上次混音 → 挂后台整个下午。
- **床头钟 / 桌面挂机**：播放时亮屏摆在床头或挂在电脑副屏，大字时钟 + 缓慢生成式画面，相当于一个氛围时钟。该场景成立依赖沉浸模式与屏幕常亮（F-9），当前尚未实现。

### 2.3 平台优先级

移动端浏览器（iOS Safari / Android Chrome）**优先于**桌面端——睡眠场景决定的。当前 v0.1 恰好在 iOS 上有两个致命缺陷（见 4.1），这是本 PRD 优先级排序的主要依据。

## 3. 现状基线（v0.1 → v0.2-rc）

以下功能已上线，后续需求不得回退这些能力：

- **26 种声音 / 8 类**（噪音、雨、水、林、风、室内、暖、冥想；v1.3 由 20 增至 26，见下方 F-7），任意叠加，每层独立音量。
- **混合音频引擎**：3 种自然声（森林/火车/咖啡馆）用真实 CC0 录音（F-1 后统一走 `fetch` + `decodeAudioData` → `AudioBufferSourceNode`，不再用 `<audio>` 元素），其余全部程序合成（Web Audio：噪声源 + 滤波 + LFO + 混响；雷/鸟/篝火/碰杯/轨道声/翻书/滴答/颂钵为事件调度合成）；无音频文件的声音自动回退合成。
- **生成式场景**：单 canvas 按各层实时音量聚合「气象强度」逐帧作画；10+ 种画法（雨丝、海浪、浮尘、风线、余烬、光斑、轨迹线、涟漪、噪点、闪电）。
- **睡眠定时**：15/30/45/60/90 分钟，结束前 20 秒渐隐。
- **床头时钟 + 混音诗行**；深/浅主题；`prefers-reduced-motion` / `prefers-color-scheme` 适配。
- **记忆**：主音量、每声音音量、主题（localStorage）。
- 基础无障碍：aria 标签、focus-visible、键盘可达。

### v0.2-rc 新增（`feat/prd-engine-lockscreen-pwa` 分支，2026-07-16）

- **F-1 引擎统一**：真实录音 `fetch` + `decodeAudioData` 进 Web Audio 图（按需懒加载解码）；file:// 下失败优雅回退合成并 toast 提示。
- **F-2（部分）**：Media Session（锁屏媒体卡片）+ 静音 `<audio>` 保活 + 后台停画；**系统中断后的自动恢复未实现（D7）**。
- **F-4 预设**：6 个内置预设 + 自定义预设（上限 12，localStorage）。
- **F-5（部分）**：会话恢复「继续上次」、页面隐藏时停画停钟、无高频动效时降帧至 ~30fps；**停播熄火未实现（D8）**。
- **F-6 PWA**：manifest + Service Worker（核心壳 network-first、音频/图标 cache-first）+ 全套图标。
- **F-8 分享**：混音（含各层音量）编码进 URL hash，实时写入地址栏。
- LICENSE（MIT）补齐、`analyser` 死代码清除（D6 关闭）。

### v0.2-rc 后 · 电台化改造（`feat/radio-tuner` 分支，2026-07-17）

- **F-4 预设 → 调频拨盘**：混音预设从 chips 升级为一台 FM 拨盘（87.5–108 MHz）。内置 6 预设固定在频道刻度（88.1 雨夜 / 90.4 晨林 / 92.9 海边 / 95.6 炉边 / 98.3 车厢 / 100.9 深空）；自定义频道可拖动改频率、可移除（上限 12→8）。
- **调台反馈**：拖动指针扫台时叠加白噪嘶声（独立于 master、静默也可听、靠近频道渐低）+ 全屏雪花噪点；松手吸附到最近频道。匹配当前混音的频道自动点亮，手动混音时指针半透明并显示「手动调音」。
- **可达性**：拨盘 `role=slider`，方向键切台、Home/End 到首尾频道；「移除此频道」为真实 `<button>`（清偿 D9 中「删除按钮键盘不可达」子项）。
- **框架化**：产品从「声音气象台」重定位为「声音电台 · Night FM」；Media Session 标题与混音诗行改用频道名。生成式气象场景与既有 20 声音/8 类保持不变。

### v1.2 后 · 挂机化 + 声音扩充 + 版权合规（`feat/immersive-lockscreen-audio` 分支，2026-07-17）

- **F-9 沉浸挂机模式**（全部实现，代码级完成、待真机验收）：播放中 8s 无操作淡出控制台与光标、时钟放大至 `20vmin`；`F` 全屏；Screen Wake Lock（仅「播放且可见」持有，切后台/暂停释放、回前台重拉，不支持时一次性提示）；睡眠定时启动后画面渐暗至约 30%（入睡调光遮罩）；时钟每分钟微漂移防烧屏；粒子密度按屏幕面积缩放（≤3× 上限）；快捷键 `Space`/`F`/`Esc`/`1–6`；点击时钟切换秒针（持久化）。
- **F-7 声音库扩充**（原 P2 提前实现）：新增 6 种 CC0 合成声音——落雪(`snow`)、地铁(`subway`)、翻书(`pages`)、滴答(`tick`)、猫呼噜(`purr`)、虫鸣(`insects`)，声音库 20→26、8 类不变，均有 canvas 画法（落雪为专属 `drawSnow` 粒子，虫鸣借 `bokeh` 萤火光斑）。
- **细分子选择器**（F-7 子项，通用 `VARIANT_PICKERS`）：把一个声音再拆成可勾选的成员，分类底部内嵌一排 chips——鸟鸣·鸟种（麻雀/画眉/斑鸠/杜鹃/夜莺）、虫鸣·虫种（蟋蟀/蝉/蛙/纺织娘）、篝火·爆裂性格（噼啪/爆裂/塌陷/余烬）、咖啡馆·环境细节（碰杯/磨豆/咖啡机/汤匙）为多选合唱；雨·打在什么上（屋檐/窗玻璃/树叶/水面）为单选，实时切换末级滤波、对小雨/大雨/雷雨通用。选中即开对应声音、随其推子发声、持久化（`lull.birds`/`insects`/`fire`/`cafe`/`rainsurf`）、多选组至少留一种（咖啡馆可全关只留底噪录音）。旧单振荡器 `chirp` 已重写为分物种拟真合成（消除鸟叫「太假」问题）。
- **F-3 版权合规**：移除 5 个 CC BY-SA 素材改用 CC0 合成，发布集全 CC0/可商用；固化素材流水线 `scripts/prep-audio.mjs`（单声道 → 首尾交叉淡化 → loudnorm → mp3）+ 候选调研助手 `scripts/fetch-cc0.mjs`（按许可证机器核验）。
- **D7/D8 补齐**：中断自愈（`AudioContext.onstatechange` + 保活元素 `pause`，每 1.5s 重试）、停播 0.9s 后 `ctx.suspend()` 熄火 DSP（D8 尚有 timer 空转残留，见台账）。

## 4. 需求详述

优先级定义：**P0** = 不做则产品在主场景不可用/不可发布；**P1** = 显著提升核心体验；**P2** = 增强项。

### 4.1 P0 — 音频引擎统一与移动端可用性（v0.2「引擎版」）

#### F-1 真实音频统一进 Web Audio 图

**需求**：废弃 `<audio>` 元素路径，真实录音改为 `fetch` + `decodeAudioData` 解码进 `AudioBufferSourceNode`（loop=true），与合成层共用同一套 gain/混响/主总线。

**理由**：一次改动同时消灭 D1（代码去重）、D2（iOS 音量）、D3（AudioBufferSourceNode 循环为采样级无缝，roadmap 原「双元素交叉淡化」方案作废）。素材仅 60s 单声道 mp3（约 0.5MB/个），解码内存开销可接受。

**验收标准**：
- [ ] 代码中不再存在 `l.real` 分叉；`rampEl`/`effReal` 删除。
- [ ] iOS Safari 上拖动任意真实录音层的推子，音量实时变化。
- [ ] 任意真实录音单独播放 5 分钟，循环点无可闻「咔哒」或空隙（如素材首尾不齐，允许对素材做首尾交叉淡化预处理）。
- [ ] 定时器渐隐对所有层（合成+真实）统一生效。
- [ ] 双击 `index.html`（file:// 协议）时 fetch 失败需优雅回退到合成，不白屏、不静默失败（提示或自动降级）。

**开放问题**：file:// 下 fetch 受限 → 方案 A：检测失败回退合成并提示「本地直开模式仅合成音」；方案 B：构建时把 mp3 转 base64 内联（单文件哲学的极致，但文件膨胀至 ~5MB）。倾向 A，v1.0 的 PWA（F-6）会让 file:// 场景边缘化。

#### F-2 锁屏存活（Media Session + 保活）

**需求**：手机锁屏/切后台后声音持续播放；锁屏界面显示媒体控制（播放/暂停、当前混音名）。

**方案要点**：
- 接入 Media Session API：metadata 用当前混音诗行（如「雨落如帘」）+ 产品名；响应 play/pause。
- iOS 保活：保持一个播放中的静音 `<audio>` 元素（或等效手段）防止 AudioContext 被挂起；`visibilitychange`/`resume` 兜底恢复。
- 锁屏/后台时暂停 canvas 渲染（rAF 自动节流，但事件调度器继续跑声音）。
- **中断恢复（v1.1 新增，D7）**：来电/闹钟/Siri 等系统中断会暂停保活元素、把 AudioContext 置为 interrupted；只靠 `visibilitychange` 恢复意味着必须解锁亮屏，睡着的用户不会解锁。需监听 `AudioContext.onstatechange` 与保活元素的 `pause` 事件，在中断结束后自动重拉播放。
- 保活逻辑与 Media Session 解耦：`keepAlive` 不得因 `mediaSession` 特性检测缺失而被跳过（D9）。

**验收标准**：
- [ ] iOS Safari：播放中锁屏 30 分钟，声音不间断，定时器到点仍正常渐隐停止。
- [ ] Android Chrome：同上，且锁屏界面出现媒体卡片可暂停/恢复。
- [ ] 解锁回到页面，UI 状态（播放态、倒计时）与实际一致。
- [ ] **（v1.1 新增）**播放中被来电/闹钟中断，中断结束后 ≤5s 自动恢复播放，无需解锁或亮屏。

#### F-3 素材版权合规

**需求**：替换 5 个 CC BY-SA 素材（小雨、雷雨、瀑布、鸟鸣、大风）为 CC0/可商用录音；补齐 5 个尚无真实录音的自然声（大雨、海浪、溪流、篝火、微风）。来源：Pixabay / Freesound-CC0。所有素材要求：可无缝循环（或预处理为可循环）、单声道、≥60s、响度归一。

### 4.2 P1 — 核心体验提升（v0.3「体验版」）

#### F-4 混音预设 → 电台频道（调频拨盘）

**需求**：混音预设以「电台频道」形式呈现，主控件为复古调频拨盘（FM 87.5–108 MHz）。内置 6 精选频道固定在刻度上（88.1 雨夜 / 90.4 晨林 / 92.9 海边 / 95.6 炉边 / 98.3 车厢 / 100.9 深空）；支持把当前混音存为自定义频道（localStorage，含各层音量与频率），可拖动改频率、可移除。拖动指针扫台时以白噪嘶声 + 雪花噪点反馈，靠近频道吸附。

**验收标准**：
- [x] 频道一键切换有整体交叉淡化，无声音断崖（沿用 `loadMix` 淡入）。
- [x] 自定义频道可存、可删、可覆盖，刷新后保留；频道数量上限 8。
- [x] 空状态时拨盘即为首屏入口，指针停在最低频、显示「静默 · 未调频」（降低「20 个声音从哪开始」的选择成本）。
- [x] 匹配当前混音的频道自动点亮；手动调音（不匹配任何频道）时指针半透明、显示「手动调音」。
- [ ]（待真机验收）拨盘键盘可达且不与走带/推子操作冲突；调台嘶声在锁屏/后台不误触发、随暂停熄火（依赖 D8）。

#### F-5 播放连续性细节

- 会话恢复：记住上次的混音组合，重开页面一键「继续上次」（不自动播放，尊重浏览器策略）。
- 页面隐藏时降载：canvas 停画、时钟停跳，仅保留音频调度。
- 长时间播放稳定性：连续播放 8 小时无内存增长失控（事件调度器创建的节点必须可靠回收——现有 `onended` 清理模式保持）。
- **停播即熄火（v1.1 新增，D8）**：暂停或定时结束后音频图不得空转——挂起 AudioContext（或拆除各层），CPU/电耗回落至空载水平。定时器把用户送入睡眠后，引擎不应再烧一整夜的电。

**验收标准**：
- [ ] 桌面 Chrome 连播 8 小时，JS 堆增长 < 20MB。
- [ ] 亮屏播放 1 小时（床头钟场景），canvas 帧率自适应降至 ≤30fps 时视觉无明显劣化（提供实现选项：无高频动效声音时降帧）。
- [ ] **（v1.1 新增）**暂停或定时结束 60s 后，CPU 占用回落至「页面打开但未播放」的水平；从暂停态恢复播放无可感知延迟。**（v1.3 现状：仅 DSP 已 `ctx.suspend()`，事件调度器 `setTimeout` 仍空转，CPU 未完全回落——须先补齐 D8 再验收。）**

#### F-9 沉浸挂机模式（v1.1 评审新增）

> 背景：挂机（桌面副屏专注 / 床头钟）是产品的第二主场景，但视觉层从未为它设计——控制台常驻遮挡场景、无全屏、无屏幕常亮、无防烧屏。生成式场景是产品最大的差异化，挂机时它就是产品的橱窗，不该永远被一块操作台挡着。

**需求**：

- **沉浸态**：播放中无交互 8 秒后，控制台与光标淡出，仅保留场景 + 时钟；时钟放大至约 `20vmin` 级别（现上限 116px 在大屏上过小）；任何输入 300ms 内恢复完整 UI。暂停时不进入沉浸态。
- **全屏**：头部提供全屏按钮（`requestFullscreen`），快捷键 `F`。
- **屏幕常亮（Wake Lock）**：播放且页面可见时申请 Screen Wake Lock，暂停/切后台时释放；不支持时降级为一次性提示。没有它，手机床头钟场景 30 秒即被系统熄屏杀死。
- **入睡调光**：睡眠定时器启动后，画面与 UI 亮度在数分钟内渐降至约 30%——声音渐隐的视觉对应物，「睡眠优先」原则的补全。
- **防烧屏**：沉浸态下无固定位置的高对比常亮像素——时钟每分钟漂移数像素，高亮元素（`--warm` 色）不长驻。
- **气象密度一致性**：canvas 粒子数按屏幕面积缩放（设上限防性能失控）。现固定粒子数（雨 200 / 星 80 / 风 70）在 4K 大屏上「大雨看起来像毛毛雨」，破坏「画面是混音的镜像」这一核心概念。
- **桌面快捷键**：`Space` 播放/暂停、`F` 全屏、`Esc` 退出沉浸、`1–6` 切内置预设。
- 秒针显示可切换（点击时钟切换，状态记忆）——挂机时的视觉噪音与烧屏点。

**实现状态（v1.3，代码级完成 · 待真机验收）**：本节全部能力已在 `feat/immersive-lockscreen-audio` 分支实现——沉浸态 8s 隐藏 UI + 光标、时钟放大 `20vmin`、~220ms 恢复、暂停不进入；全屏（按钮 + `F` + webkit 兜底 + aria 反映）；Wake Lock（仅播放且可见持有、暂停/后台释放、回前台重拉、不支持一次性提示）；入睡调光遮罩（`prefers-reduced-motion` 直达目标）；时钟每分钟 ±7/±5px 漂移防烧屏；粒子按屏面积缩放（0.7–3×、分型再设子上限）；快捷键 `Space`/`F`/`Esc`/`1–6`；秒针可切换并持久化。**两处代码级偏差记入 D11**：① `F`/`1–6` 在 `<button>` 聚焦时未加守卫（仅 `Space` 有）；② 沉浸态 `datemeta` 日期行仍可见。

**验收标准**：
- [ ] 桌面：播放中 8s 无输入进入沉浸态（控制台与光标淡出）；移动鼠标/触摸后 300ms 内恢复，全程无布局跳动。
- [ ] 手机床头：播放且亮屏时屏幕不自动熄灭（iOS Safari 16.4+ / Android Chrome）；暂停或切后台后锁释放。
- [ ] OLED 屏连续挂机 8 小时无残影风险点：时钟漂移生效、无静止高对比元素。
- [ ] 定时器启动后画面渐暗至约 30%；`prefers-reduced-motion` 下跳过动画直接到目标亮度。
- [ ] 27 寸 4K 与 6 寸手机同放「大雨 70%」，雨丝密度主观一致。
- [ ] 快捷键不与输入控件冲突（焦点在 range/button 上时不误触发）；沉浸态不破坏键盘可达性与 aria。

### 4.3 P2 — 增强（v1.0「发布版」及以后）
