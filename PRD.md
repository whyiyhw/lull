## 1. 产品概述

### 1.1 一句话定位

**Lull 是一个「声音气象台」：白噪音混音助眠/专注网页应用，你叠加什么声音，整屏画布就实时生长出对应的气象。**

### 1.2 背景

白噪音/环境音应用是成熟品类（Noisli、A Soft Murmur、moodist、潮汐等），但普遍是「一排开关 + 一张静态背景图」。Lull 的差异化是**混音驱动的生成式场景**：雨声带来雨丝、海浪带来潮汐波带、鸟鸣迸出金色火花、雷雨触发闪电——画面是混音的镜像，不是装饰。

同时坚持**本地优先**的工程哲学：纯前端、零运行时依赖、单文件可双击运行、无账号、无埋点、数据只存 localStorage。

### 1.3 设计原则（约束所有需求）

1. **睡眠优先**：任何功能不得引入影响入睡的元素（突兀音量跳变、亮屏干扰、循环接缝）。
2. **本地优先**：无后端、无账号、无第三方请求；离线可用是目标而非选配。
3. **单文件克制**：运行时零依赖；构建工具只服务开发体验，产物仍是可双击打开的静态文件。
4. **版权洁癖**：所有音频素材必须 CC0/可商用，逐文件登记于 `AUDIO_CREDITS.md`。

## 2. 目标用户与场景

### 2.1 用户画像

| 画像 | 描述 | 核心诉求 |
|---|---|---|
| 睡眠困难者（主） | 入睡慢、易被环境音吵醒，睡前手机放床头 | 稳定不间断的声音、定时关闭、锁屏后继续播放、暗色不刺眼 |
| 专注工作者（次） | 开放办公室/在家办公，用环境音屏蔽干扰 | 快速调出熟悉的混音、长时间播放不疲劳、不占心智 |
| 冥想/放松用户（次） | 短时段放松、呼吸练习 | 颂钵等冥想音色、画面氛围感 |

### 2.2 核心场景

- **睡前**（主场景，移动端为主）：躺床上打开 → 调出「雨夜」混音 → 设 30 分钟定时 → 锁屏 → 声音持续、到点渐隐停止。
- **专注**（桌面为主）：开工时一键恢复上次混音 → 挂后台整个下午。
- **床头钟**：播放时亮屏摆在床头，大字时钟 + 缓慢生成式画面，相当于一个氛围时钟。

### 2.3 平台优先级

移动端浏览器（iOS Safari / Android Chrome）**优先于**桌面端——睡眠场景决定的。当前 v0.1 恰好在 iOS 上有两个致命缺陷（见 4.1），这是本 PRD 优先级排序的主要依据。

## 3. 现状基线（v0.1 已实现）

以下功能已上线，后续需求不得回退这些能力：

- **20 种声音 / 8 类**（噪音、雨、水、林、风、室内、暖、冥想），任意叠加，每层独立音量。
- **混合音频引擎**：8 种自然声用真实录音（`<audio loop>`），其余程序合成（Web Audio：噪声源 + 滤波 + LFO + 混响；雷/鸟/篝火/碰杯/轨道声/颂钵为事件调度合成）；无音频文件的声音自动回退合成。
- **生成式场景**：单 canvas 按各层实时音量聚合「气象强度」逐帧作画；10+ 种画法（雨丝、海浪、浮尘、风线、余烬、光斑、轨迹线、涟漪、噪点、闪电）。
- **睡眠定时**：15/30/45/60/90 分钟，结束前 20 秒渐隐。
- **床头时钟 + 混音诗行**；深/浅主题；`prefers-reduced-motion` / `prefers-color-scheme` 适配。
- **记忆**：主音量、每声音音量、主题（localStorage）。
- 基础无障碍：aria 标签、focus-visible、键盘可达。

### 已知技术债（评审结论，驱动 P0 需求）

| # | 问题 | 影响 |
|---|---|---|
| D1 | 真实录音走 `<audio>` 元素，绕过 Web Audio 图 | 音量/渐变/定时逻辑双份实现；无法过混响总线；引出 D2/D3 |
| D2 | iOS Safari 忽略 `HTMLMediaElement.volume` 写入 | iPhone 上 8 个真实录音层不受音量推子控制、渐隐失效 |
| D3 | `<audio loop>` 循环有可听接缝 | 睡眠场景的头号质量杀手 |
| D4 | 锁屏后 AudioContext 被挂起，合成层停播而 `<audio>` 层继续 | 锁屏瞬间混音残缺；睡眠主场景不可用 |
| D5 | 5 个音频素材为 CC BY-SA（copyleft），不可商用 | 阻塞正式发布 |
| D6 | `analyser` 死代码；README 声称 MIT 但无 LICENSE 文件 | 卫生问题 |

## 4. 需求详述

优先级定义：**P0** = 不做则产品在主场景不可用/不可发布；**P1** = 显著提升核心体验；**P2** = 增强项。

### 4.1 P0 — 音频引擎统一与移动端可用性（v0.2「引擎版」）

#### F-1 真实音频统一进 Web Audio 图

**需求**：废弃 `<audio>` 元素路径，真实录音改为 `fetch` + `decodeAudioData` 解码进 `AudioBufferSourceNode`（loop=true），与合成层共用同一套 gain/混响/主总线。

**理由**：一次改动同时消灭 D1（代码去重）、D2（iOS 音量）、D3（AudioBufferSourceNode 循环为采样级无缝，roadmap 原「双元素交叉淡化」方案作废）。素材仅 60s 单声道 mp3（约 0.5MB/个），解码内存开销可接受。

**验收标准**：
- [ ] 代码中不再存在 `l.real` 分叉；`rampEl`/`effReal` 删除。
- [ ] iOS Safari 上拖动任意真实录音层的推子，音量实时变化。
- [ ] 任意真实录音单独播放 5 分钟，循环点无可闻「咔哒」或空隙（如素材首尾不齐，允许对素材做首尾交叉淡化预处理）。
- [ ] 定时器渐隐对所有层（合成+真实）统一生效。
- [ ] 双击 `index.html`（file:// 协议）时 fetch 失败需优雅回退到合成，不白屏、不静默失败（提示或自动降级）。

**开放问题**：file:// 下 fetch 受限 → 方案 A：检测失败回退合成并提示「本地直开模式仅合成音」；方案 B：构建时把 mp3 转 base64 内联（单文件哲学的极致，但文件膨胀至 ~5MB）。倾向 A，v1.0 的 PWA（F-6）会让 file:// 场景边缘化。

#### F-2 锁屏存活（Media Session + 保活）

**需求**：手机锁屏/切后台后声音持续播放；锁屏界面显示媒体控制（播放/暂停、当前混音名）。

**方案要点**：
- 接入 Media Session API：metadata 用当前混音诗行（如「雨落如帘」）+ 产品名；响应 play/pause。
- iOS 保活：保持一个播放中的静音 `<audio>` 元素（或等效手段）防止 AudioContext 被挂起；`visibilitychange`/`resume` 兜底恢复。
- 锁屏/后台时暂停 canvas 渲染（rAF 自动节流，但事件调度器继续跑声音）。

**验收标准**：
- [ ] iOS Safari：播放中锁屏 30 分钟，声音不间断，定时器到点仍正常渐隐停止。
- [ ] Android Chrome：同上，且锁屏界面出现媒体卡片可暂停/恢复。
- [ ] 解锁回到页面，UI 状态（播放态、倒计时）与实际一致。

#### F-3 素材版权合规

**需求**：替换 5 个 CC BY-SA 素材（小雨、雷雨、瀑布、鸟鸣、大风）为 CC0/可商用录音；补齐 5 个尚无真实录音的自然声（大雨、海浪、溪流、篝火、微风）。来源：Pixabay / Freesound-CC0。所有素材要求：可无缝循环（或预处理为可循环）、单声道、≥60s、响度归一。

**验收标准**：
- [ ] `AUDIO_CREDITS.md` 全表许可证为 CC0/Public domain/可商用，无 ⚠️ 行。
- [ ] 各声音之间主观响度均衡（同音量推子下无某声音明显压过其他）。
- [ ] 补充 LICENSE 文件（代码 MIT），消除 D6。

### 4.2 P1 — 核心体验提升（v0.3「体验版」）

#### F-4 混音预设

**需求**：内置精选预设一键调用（首批 6 个：雨夜、晨林、海边、炉边、车厢、深空/纯噪），并支持把当前混音存为自定义预设（localStorage，含各层音量）。

**验收标准**：
- [ ] 预设一键切换有整体交叉淡化，无声音断崖。
- [ ] 自定义预设可存、可删、可覆盖，刷新后保留；预设数量上限 12。
- [ ] 首屏引导：空状态时展示预设入口（降低「20 个声音从哪开始」的选择成本）。

#### F-5 播放连续性细节

- 会话恢复：记住上次的混音组合，重开页面一键「继续上次」（不自动播放，尊重浏览器策略）。
- 页面隐藏时降载：canvas 停画、时钟停跳，仅保留音频调度。
- 长时间播放稳定性：连续播放 8 小时无内存增长失控（事件调度器创建的节点必须可靠回收——现有 `onended` 清理模式保持）。

**验收标准**：
- [ ] 桌面 Chrome 连播 8 小时，JS 堆增长 < 20MB。
- [ ] 亮屏播放 1 小时（床头钟场景），canvas 帧率自适应降至 ≤30fps 时视觉无明显劣化（提供实现选项：无高频动效声音时降帧）。

### 4.3 P2 — 增强（v1.0「发布版」及以后）

#### F-6 PWA

manifest + service worker，可安装到主屏、完全离线使用（含音频素材缓存）。这是「本地优先」哲学的完成态，也化解 F-1 的 file:// 开放问题。
**验收**：断网状态下从主屏图标冷启动，全功能可用。

#### F-7 声音库扩充

下雪静谧、地铁、翻书、时钟滴答、猫呼噜等；每个新声音必须同时提供 paint 气象画法（保持「气象台」概念完整性），且遵守 F-3 的素材标准。

#### F-8 分享混音

把当前混音编码进 URL hash（如 `#lrain=60&fire=40`），打开链接即还原混音。无后端，纯前端解析。

#### 明确不做（Non-goals）

- ❌ 账号体系、云同步、社区/UGC——违背本地优先原则。
- ❌ 原生 App——PWA 覆盖需求；如未来做，另立项目。
- ❌ 埋点/分析 SDK——隐私原则；效果验证靠 GitHub star/issue 反馈与自用。
- ❌ AI 生成音频——现阶段合成器 + 精选录音已足够，不为概念加复杂度。

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 兼容性 | iOS Safari ≥ 16、Android Chrome ≥ 110、桌面 Chrome/Edge/Firefox/Safari 近两年版本 |
| 性能 | 首屏可交互 < 1s（本地/缓存）；音频解码总内存 < 80MB；亮屏播放整机 CPU < 15%（中端手机） |
| 电池 | 锁屏播放 8 小时耗电应与系统音乐播放同数量级（canvas 必须停画） |
| 无障碍 | 全键盘可达；aria 完备；`prefers-reduced-motion` 下无动效但功能完整 |
| 隐私 | 零网络请求（素材本地/同源）、零 Cookie、仅 localStorage |
| 体积 | 页面代码（不含音频）gzip 后 < 60KB；音频素材总量 < 15MB |
| 代码结构 | v0.2 起允许拆分 ES modules（Vite 构建仍输出自包含产物），单文件 814 行已到可维护边界 |

## 6. 版本规划

```
v0.1 ──────► v0.2 引擎版 ──────► v0.3 体验版 ──────► v1.0 发布版
(现状)        F-1 引擎统一          F-4 预设             F-6 PWA
              F-2 锁屏存活          F-5 连续性            F-7 更多声音
              F-3 素材合规                               F-8 分享
              ↑ 全部完成才有                              ↑ 完成后可
                "能睡着觉的最小产品"                        对外正式发布
```

**里程碑判定**：
- **v0.2 = MLP（Minimum Lovable Product）**：一个 iPhone 用户能完整走通「选混音 → 定时 → 锁屏入睡 → 自动停止」，这是产品存在意义的最小闭环。
- **v1.0 = 可对外推广**：素材全部可商用、可安装、可离线、可分享。

## 7. 成功度量

无埋点前提下的替代指标：

- **定性主指标**：作者及种子用户连续 7 天真实用于入睡，无「循环接缝惊醒 / 锁屏断播 / 音量失控」任何一项发生（每项即 P0 bug）。
- **工程指标**：第 5 节非功能需求全部达标，作为发布门禁。
- **公开指标**（v1.0 后）：GitHub star / issue 中的真实使用反馈；PWA 安装无从统计，接受。

## 8. 风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| iOS 音频策略随版本变化（保活手段失效） | 高 | F-2 验收纳入每次 iOS 大版本回归清单；保活方案做成可替换模块 |
| CC0 素材质量参差，循环预处理工作量超预期 | 中 | 合成回退兜底：任何素材不达标就先用合成版发布，不阻塞版本 |
| 单人项目，P0 三项工程量集中在音频引擎 | 中 | F-1 先行（F-2/F-3 依赖其落地形态）；每项独立可发布 |
| file:// 直开体验降级引发老用户困惑 | 低 | 降级提示文案 + README 说明；PWA 到位后自然消解 |
