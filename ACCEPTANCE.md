# Lull · 真机验收清单（v0.2 发布门禁）

> **为什么存在**：PRD 把「能睡着觉的最小产品（MLP）」的唯一硬门禁定为 **F-1 + F-2（含 D7）+ F-3 全部通过真机验收**。所有功能均已代码实现且部分经 headless 验证，但**零条真机执行记录 = 纸面 done**。本清单把门禁拆成可勾选的步骤，跑一遍并回填结果，才算 done。
>
> **必须用真机 + http(s)/PWA**：`file://` 直开会因 `fetch` 受限回退合成音、且无 Service Worker/锁屏保活，**不能**用于验收。用 `pnpm dev`（同一 Wi-Fi 下手机开 `http://<你的电脑IP>:5173`）或部署到静态托管后访问。
>
> 记录格式：每项填 **设备 / 系统版本 / 日期 / ✅或❌ / 备注**。任一 ❌ 命中「P0 bug」（循环接缝惊醒 / 锁屏断播 / 音量失控三者之一）即阻塞 v0.2。

## 2. F-2 · 锁屏存活 + D7 中断自愈（**核心门禁**）

- [ ] **iOS 锁屏 30 分钟**：播放中锁屏，静置 30 分钟 → 声音**全程不间断**；定时器到点仍正常渐隐停止。
- [ ] **Android 锁屏 + 媒体卡片**：同上，且锁屏界面出现媒体卡片，可暂停 / 恢复，标题为当前频道名。
- [ ] **状态一致**：解锁回到页面，播放态 / 倒计时与实际一致，无「UI 显示暂停但仍在响」。
- [ ] **D7 中断自愈**：播放中被**来电 / 闹钟 / Siri** 打断，中断结束后 **≤5s 自动恢复播放**，无需解锁或亮屏。
  - 复现建议：iOS 播放中让另一台机打给它，接通再挂断 → 听是否自动续播。

## 4. F-5 · 停播熄火 · 电池（D8 真机数值验收）

> 代码 + headless 已证明暂停后事件调度器 `setTimeout` churn 从 13/2s 降到 2/2s（仅时钟）。此处补**整机功耗**的真机数值。

- [ ] **CPU 回落**（桌面最易测）：桌面 Chrome 播放含篝火 / 火车 / 滴答的混音，打开任务管理器看该标签页 CPU；**暂停 60s 后** CPU 回落到「页面打开但未播放」水平。
- [ ] **恢复无延迟**：从暂停恢复播放，声音无可感知延迟、无爆音。

## 5. F-9 · 沉浸挂机模式（含本批 D11 修复回归）

- [ ] **沉浸态进入 / 退出**：播放中 8s 无输入 → 控制台与光标淡出、时钟放大、日期行隐藏（**D11④**）；移动鼠标 / 触摸后 ~300ms 内恢复，无布局跳动。暂停时不进入沉浸态。
- [ ] **Wake Lock**：手机播放且亮屏时屏幕不自动熄灭（iOS 16.4+ / Android）；暂停或切后台后锁释放（不支持时有一次性提示）。
- [ ] **入睡调光**：设定时后画面数分钟内渐暗至约 30%；`prefers-reduced-motion` 下直接到位。
- [ ] **防烧屏**：OLED 挂机时钟每分钟有微漂移，无静止高对比常亮像素。
- [ ] **粒子密度一致**：同放「大雨 70%」，大屏与手机雨丝密度主观一致。
- [ ] **快捷键守卫（D11①）**：焦点在某个按钮上时按 `Space` / `F` / `1–6` **不**误触发全局动作；焦点在拨盘（slider）上时方向键只切台。`Esc` 任意时候可退出沉浸。
- [ ] **手动暂停取消定时（D11②）**：设 30 分钟定时 → 手动按播放键暂停 → 倒计时**停止**、入睡调光**归零**、定时 chip 回到「关」；不再出现「到点又触发一次暂停」。锁屏媒体卡片按暂停同样生效。
- [ ] **落雪满帧（D11③）**：单独播放「落雪」，画面为流畅连续飘雪（已入 FAST，不再掉到 ~30fps）。
